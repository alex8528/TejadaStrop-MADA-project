# Data

This folder contains datasets used in the project at different stages of the data processing workflow.

All data in this directory are loaded, cleaned, transformed, and saved using code located in the `code` directory. The goal is to ensure that all data processing steps are fully reproducible.

---

# Folder Structure

The data folder is organized into the following subfolders:

data/
│
├── raw-data
└── processed-data

### raw-data

This folder contains the **original raw data** used in the project.

For this project, the raw dataset comes from the **Pregnancy Risk Assessment Monitoring System (PRAMS) Phase 8**.

Raw data should **never be edited or modified**. These files serve as the original source data and should remain unchanged.

Instead of editing raw files directly, load them into R and perform all cleaning and transformations using code. This ensures that the entire workflow can be reproduced.

---

### processed-data

This folder contains **cleaned and analysis-ready datasets** generated from the raw data.

These datasets are created by running scripts in the `code/processing-code` folder. The processed datasets used in this project include files such as:

- `prams_clean.rds`
- `prams_analytic.rds`

Because these datasets are created by code, they should **not be manually edited**.

If needed, they can always be regenerated by rerunning the data processing scripts.

---

# Editing Raw Files

In rare cases, raw data may require manual editing before they can be imported into R. For example:

- poorly formatted Excel files
- variables encoded using formatting or colors
- inconsistent column structures

If edits are required:

1. **Create a copy of the original file**
2. Place the copy in a separate folder
3. Make edits only to the copy
4. Document all modifications made

The original raw data should always remain unchanged.

---

# Data Storage Format

Processed datasets are saved as **RDS files**. This format preserves variable types such as:

- factors
- numeric variables
- character variables

Saving processed data as `.rds` ensures that the dataset can be loaded back into R without losing important metadata.

While CSV files are easier to share, they do not preserve variable types and are therefore not used for the main analysis datasets.

---

# Reproducibility

The full workflow for data handling in this project follows the pipeline below:

raw data → processing scripts → processed data → analysis → manuscript

Running the scripts in the `code` folder will recreate the processed datasets used in the analysis.
