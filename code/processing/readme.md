# Processing Code

This folder contains scripts used to clean and process the raw PRAMS dataset for this project.

The goal of these scripts is to transform the raw data into cleaned and analysis-ready datasets that are saved in the `data/processed-data` folder.

All processing steps are performed using reproducible code so that the datasets used in the analysis can be regenerated at any time.

# Processing Workflow

The data pipeline for this project follows the structure below:

raw data → processing scripts → processed data → analysis → manuscript

Raw data are stored in:

data/raw-data/

Processed datasets generated by the scripts in this folder are saved to:

data/processed-data/

These processed datasets are then used by the analysis scripts and manuscript.

# Files in this Folder

This folder may contain multiple approaches for running the processing code.

### R Script

An R script performs the data cleaning and processing steps.  
This script typically:

- loads the raw PRAMS dataset
- recodes variables
- handles missing values
- creates analytic variables
- saves cleaned datasets to `data/processed-data`

Example output files:

prams_clean.rds  
prams_analytic.rds

### Quarto File

A Quarto (`.qmd`) file may also be used to document the data processing workflow.  
This approach allows code, explanations, and outputs to be combined in a single document.

Two common approaches are possible:

1. **All code inside the Quarto file**

   The Quarto document contains both the processing code and explanatory text.

2. **Code sourced from an R script**

   The Quarto document pulls code from an external R script and executes it.  
   This keeps the code centralized while still allowing narrative documentation.

Both approaches are valid and can be used depending on the task.

# Documentation

Processing scripts should contain sufficient comments explaining:

- variable recoding decisions
- data cleaning steps
- creation of analytic variables
- any assumptions made during processing

This documentation helps ensure the project remains understandable and reproducible.


## Reproducing the project

Because the raw PRAMS file and intermediate cleaned file may not be fully reproducible in all environments, the recommended starting point for reproduction is:

data/processed-data/prams_analytic.rds

The files `01_load_clean.qmd` and `02_create_analytic_dataset.qmd` are included for workflow documentation, but they may skip processing if required inputs are unavailable or unreadable.
